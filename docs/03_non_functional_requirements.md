# 03 – Non-Functional Requirements (Requerimientos No Funcionales)

## 1. Performance (Rendimiento)

NFR-01: The system shall respond to user actions within 3 seconds under normal load conditions.
(El sistema deberá responder a acciones del usuario en menos de 3 segundos bajo carga normal.)

NFR-02: The system shall support concurrent users without performance degradation.
(El sistema deberá soportar usuarios concurrentes sin degradación significativa del rendimiento.)

---

## 2. Security (Seguridad)

NFR-03: The system shall restrict access based on user roles and permissions.
(El sistema deberá restringir el acceso según roles y permisos.)

NFR-04: Sensitive data shall not be exposed in URL parameters.
(Los datos sensibles no deberán exponerse en parámetros de URL.)

NFR-05: The system shall enforce secure session handling.
(El sistema deberá garantizar manejo seguro de sesiones.)

---

## 3. Data Integrity (Integridad de Datos)

NFR-06: The system shall prevent duplicate appointment records.
(El sistema deberá impedir registros duplicados de turnos.)

NFR-07: The system shall maintain consistency between appointment and session records.
(El sistema deberá mantener consistencia entre turnos y sesiones médicas.)

---

## 4. Usability (Usabilidad)

NFR-08: The system shall provide clear validation error messages.
(El sistema deberá mostrar mensajes de error claros.)

NFR-09: The system interface shall be intuitive and easy to navigate.
(La interfaz deberá ser intuitiva y fácil de navegar.)

---

## 5. Reliability (Confiabilidad)

NFR-10: The system shall maintain stable operation during normal usage.
(El sistema deberá mantener funcionamiento estable bajo uso normal.)

NFR-11: The system shall not lose data in case of unexpected interruption.
(El sistema no deberá perder datos ante interrupciones inesperadas.)
