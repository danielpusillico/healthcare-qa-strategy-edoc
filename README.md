# Healthcare Appointment System – QA Strategy & Automation Portfolio  
(Sistema de Turnos Médicos – Estrategia QA y Portfolio de Automatización)

---

## Project Overview (Resumen del Proyecto)

This project demonstrates a complete end-to-end QA lifecycle applied to a healthcare appointment management system (eDoc).

(Este proyecto demuestra un ciclo completo de QA aplicado a un sistema de gestión de turnos médicos.)

The objective was to simulate a real professional QA process including requirements analysis, test design, defect management, and preparation for automation.

(El objetivo fue simular un proceso profesional real de QA incluyendo análisis de requerimientos, diseño de pruebas, gestión de defectos y preparación para automatización.)

---

## Scope of Work (Alcance del Trabajo)

The following QA activities were performed:

(Las siguientes actividades de QA fueron realizadas:)

- Functional Requirements Analysis (Análisis de Requerimientos Funcionales)
- Non-Functional Requirements Definition (Definición de Requerimientos No Funcionales)
- Test Planning (Planificación de Pruebas)
- Manual Test Case Design (Diseño de Casos de Prueba Manuales)
- Traceability Matrix Creation (Creación de Matriz de Trazabilidad)
- Defect Reporting & Severity Classification (Reporte de Defectos y Clasificación por Severidad)
- Risk Analysis (Análisis de Riesgos)
- Test Summary Reporting (Informe Final de Pruebas)
